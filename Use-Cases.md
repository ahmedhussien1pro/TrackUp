# TrackUp — Use Cases (POC)

## 1. User Doesn't Know What to Choose

**Actor:** New User

### Flow:

1. يدخل المنصة
2. يبدأ التقييم
3. يدخل بياناته
4. يعمل Pre-Test
5. يشوف Top 3 Tracks
6. يدخل Free Preview

### Outcome:

- يبدأ يفهم المجالات
- ياخد اتجاه مبدئي

---

## 2. User Knows General Field

**Actor:** Semi-Decided User

### Flow:

1. يبدأ التقييم
2. يختار "عارف المجال"
3. يختار Track (Software مثلًا)
4. يدخل Track Experience
5. يحجز Mentor أو يشوفه
6. يعمل Specialization Test

### Outcome:

- يحدد تخصص دقيق (Frontend مثلًا)

---

## 3. User Upgrades

**Actor:** Interested User

### Flow:

1. يشوف النتائج
2. يحاول يدخل التفاصيل
3. يلاقيها مقفولة
4. يعمل Upgrade
5. يرجع يكمل

### Outcome:

- يتحول لـ Paid User

---

## 4. User Uses Mentor

**Actor:** Paid User

### Flow:

1. يدخل Mentor Section
2. يشوف Mentors
3. يحجز Session أو Chat
4. يسأل عن المجال

### Outcome:

- يقلل الحيرة
- ياخد قرار أوضح

---

## 5. User Completes Roadmap

**Actor:** Active User

### Flow:

1. يدخل Profile
2. يشوف Roadmap
3. يبدأ Checklist
4. يكمل خطوات

### Outcome:

- تقدم فعلي في التعلم

---

## 6. User Returns Later

**Actor:** Returning User

### Flow:

1. يرجع بعد فترة
2. يلاقي التقدم محفوظ
3. يكمل من نفس النقطة

### Outcome:

- زيادة Retention

---

## 7. User Doesn't Upgrade

**Actor:** Free User

### Flow:

1. يشوف Preview
2. يرفض الدفع
3. يخرج

### System Behavior:

- حفظ التقدم
- عرض CTA للرجوع لاحقًا

---

## 8. User Loops Between Mentor & Learning

**Actor:** Paid User

### Flow:

1. يبدأ Roadmap
2. يقف عند نقطة
3. يرجع للـ Mentor
4. ياخد مساعدة
5. يكمل

### Outcome:

- تجربة Guided حقيقية
